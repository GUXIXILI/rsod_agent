## 二、摄像头无法连接 — 问题诊断（与讲义对比）

### 2.1 问题现象

点击"开启摄像头"按钮后，浏览器正常弹出摄像头权限授权，用户授权后按钮变为禁用状态，但没有任何画面显示。

### 2.2 控制台错误信息

```
[error] [WebSocket] 连接错误:
[info] [WebSocket] 连接已关闭
```

### 2.3 后端日志确认

```
WARNING  | app.api.camera:camera_websocket:48 | WebSocket 连接被拒绝：缺少 token
```

### 2.4 逐问题对比分析（讲义 vs 实际代码）

以下逐一对比讲义原始代码与实际工程代码，确认问题所在。

---

#### 问题 1：Token 存储 key 不一致（P0，确认存在）

**讲义代码（Day03 store 设计）：**

文件：`src/stores/user.js`，讲义第 684-722 行

```javascript
const TOKEN_KEY = 'rsod_token'
const USER_KEY = 'rsod_user'

export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem(TOKEN_KEY) || '',
    user: JSON.parse(localStorage.getItem(USER_KEY) || 'null'),
  }),
  actions: {
    async login(credentials) {
      const res = await loginApi(credentials)
      this.token = res.access_token
      localStorage.setItem(TOKEN_KEY, res.access_token)  // key = 'rsod_token'
      // ...
    },
  },
})
```

**实际代码：**

文件：`frontend/src/stores/user.js`，第 60 行

```javascript
localStorage.setItem('token', access_token)  // key = 'token'（与讲义不同）
```

**影响分析：**

讲义使用 `rsod_token` 作为统一 key，实际 store 实现时被改成了 `token`。摄像头页面（CameraDetectionPage.vue 第 286 行）仍然按照讲义读取 `localStorage.getItem("rsod_token")`，导致取到 `null`。

---

#### 问题 2：WebSocket 连接未传递 token（P0，确认存在）

**讲义代码（Day09 后端）：**

文件：`app/api/detection.py`，讲义第 1293-1308 行

```python
@router.websocket("/camera")
async def camera_detection_ws(websocket: WebSocket):
    """摄像头实时检测 WebSocket 接口"""
    await websocket.accept()  # 直接接受，无 JWT 认证
    # ...
```

**实际代码：**

文件：`backend/app/api/camera.py`，第 29-48 行

```python
@router.websocket("/camera")
async def camera_websocket(websocket: WebSocket, token: str = ""):
    """摄像头实时检测 WebSocket 接口"""
    if not token:
        await websocket.close(code=1008, reason="missing token")
        return
    # 验证 JWT Token
    payload = verify_token(token)
    if not payload:
        await websocket.close(code=1008, reason="invalid token")
        return
    await websocket.accept()
```

**讲义代码（Day09 前端）：**

文件：`frontend/src/utils/cameraWs.js`，讲义第 2600-2604 行

```javascript
const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
const host = window.location.host;
const wsUrl = `${protocol}//${host}/api/detection/camera`;
this.ws = new WebSocket(wsUrl);  // 无 token 参数
```

**实际代码：**

文件：`frontend/src/views/CameraDetectionPage.vue`，第 279-288 行

```javascript
const protocol = window.location.protocol === "https:" ? "wss:" : "ws:";
const host = window.location.host;
const wsUrl = `${protocol}//${host}/api/detection/camera`;
const token = localStorage.getItem("rsod_token");  // 获取了 token 但未使用
ws = new WebSocket(wsUrl);  // 未将 token 传入 URL
```

**影响分析：**

讲义的后端 WebSocket 没有 JWT 认证，前端也不需要传 token。但实际后端在实现时增加了 JWT 认证（要求 URL query 参数 `token`），而前端代码直接复制自讲义，没有在 URL 中附加 token。后端因缺少 token 直接关闭连接（code=1008）。

#### 问题 3：配置确认消息类型不一致（P1，确认存在）

**讲义代码（Day09 后端）：**

case 'config_ok':

#### 问题 4：cameraWs.js 工具文件缺失（P2，新发现）

**讲义代码（Day09）：**

讲义第 2549-2728 行提供了完整的 `frontend/src/utils/cameraWs.js` 文件，封装了 `CameraWs` 类，提供 `createCameraWs()` 工厂函数，包含连接管理、帧发送、消息处理、重连等逻辑。

**实际代码：**

`frontend/src/utils/cameraWs.js` 文件不存在。摄像头 WebSocket 的逻辑直接内嵌在 `CameraDetectionPage.vue` 的 `<script setup>` 中，没有复用封装。

**讲义设计意图：**

讲义将 WebSocket 逻辑抽离到独立工具类，前端组件通过 `createCameraWs()` 调用，职责分离清晰。实际代码将所有逻辑内嵌在组件中，违反了讲义的模块化设计。

#### 问题 5：后端 WebSocket 路径与认证策略与讲义不同（P2，新发现）

| 对比维度 | 讲义 | 实际 |
|---------|------|------|
| 后端文件 | `app/api/detection.py`（在检测路由中追加） | `app/api/camera.py`（独立文件） |
| 路由路径 | `/camera` | `/api/detection/camera` |
| 认证方式 | 无 JWT 认证 | 有 JWT 认证（token query 参数） |
| 模型加载 | 调用 `get_model(scene_id)` | 调用 `FireSmokeDetectionService` |

**影响分析：**

讲义后端无 JWT 认证，实际后端增加了 JWT 认证。这是一个安全改进，但导致了前端 WebSocket 连接失败（问题 2），因为前端未适配这个变更。

---

#### 问题 6：DetectionPage 与 CameraDetectionPage 双版本共存（P2，新发现）

**讲义设计：**

讲义 Day09 将摄像头检测功能直接实现在 `DetectionPage.vue` 中（替换原有上传检测页面），讲义第 2732-3117 行提供了完整的 `DetectionPage.vue` 代码。

**实际代码：**

保留了两个独立页面：
- `/detection` → `DetectionPage.vue`（上传检测，无摄像头功能）
- `/camera-detection` → `CameraDetectionPage.vue`（摄像头检测，独立页面）

**说明：**

这是项目"只增不减"策略的体现，属于有意的双版本共存。但 `DetectionPage.vue` 中完全没有摄像头检测功能，与讲义期望不符。

## 三、解决方案（参考讲义）

以下方案以讲义代码为权威参考，修复当前代码与讲义之间的偏差。

### 方案一：修复 Store 的 Token Key（修复问题 1）

**目标：** 使 Token 存储 key 与讲义保持一致。

**修改文件：** `frontend/src/stores/user.js`

**修改内容：** 将 store 中的 token key 从 `'token'` 改回 `'rsod_token'`，与讲义 Day03 保持一致。

讲义 Day03 代码（第 684-722 行）中 `TOKEN_KEY = 'rsod_token'`，所有存储和读取均使用此 key。实际 store 应与此保持一致。

同时需要全局检查所有使用 `localStorage.getItem('token')` 和 `localStorage.setItem('token', ...)` 的地方，统一改为 `rsod_token`。

**涉及文件：**
- `frontend/src/stores/user.js`（第 60 行，setItem 的 key）
- `frontend/src/stores/user.js`（state 初始化，getItem 的 key）
- `frontend/src/utils/request.js`（请求拦截器中读取 token 的 key）
- `frontend/src/router/index.js`（路由守卫中读取 token 的 key）

---

### 方案二：修复后端 WebSocket 认证 + 前端 Token 传递（修复问题 2）

**选项 A（推荐，与讲义保持一致）：后端移除 JWT 认证**

**目标：** 使后端 WebSocket 行为与讲义完全一致。

**修改文件：** `backend/app/api/camera.py`

**修改内容：** 将后端 WebSocket 改为与讲义相同的方式，直接 `await websocket.accept()`，移除 JWT 认证逻辑。

讲义 Day09 后端代码（第 1293-1308 行）：

```python
@router.websocket("/camera")
async def camera_detection_ws(websocket: WebSocket):
    await websocket.accept()  # 直接接受，无 JWT 认证
    connection_id = id(websocket)
    # ...
```

**选项 B（保留 JWT 认证，前端适配）：前端传递 token**

**目标：** 保留后端的 JWT 认证安全改进，同时修复前端。

**修改文件：** `frontend/src/views/CameraDetectionPage.vue`（第 283-288 行）

**修改内容：** 将 token 附加到 WebSocket URL 的 query 参数中。

修改前（第 283-288 行）：

```javascript
const wsUrl = `${protocol}//${host}/api/detection/camera`;
const token = localStorage.getItem("rsod_token");
ws = new WebSocket(wsUrl);
```

修改后：

```javascript
const token = localStorage.getItem("rsod_token");
const wsUrl = `${protocol}//${host}/api/detection/camera?token=${encodeURIComponent(token)}`;
ws = new WebSocket(wsUrl);
```

---

### 方案三：统一配置确认消息类型（修复问题 3）

**目标：** 使前后端消息类型一致。

**选项 A（推荐，与讲义保持一致）：后端改为 `config_ok`**

**修改文件：** `backend/app/api/camera.py`（第 139 行）

**修改内容：** 将 `"type": "config_ack"` 改为 `"type": "config_ok"`。

讲义 Day09 后端代码（第 1365-1371 行）使用 `config_ok`，前端也按讲义使用 `config_ok`。

**选项 B：前端改为 `config_ack`**

**修改文件：** `frontend/src/views/CameraDetectionPage.vue`（第 316 行）

**修改内容：** 将 `data.type === "config_ok"` 改为 `data.type === "config_ack"`。

---

### 方案四：创建 cameraWs.js 工具文件（修复问题 4）

**目标：** 按讲义创建 WebSocket 封装工具类，使组件专注于页面渲染。

**新建文件：** `frontend/src/utils/cameraWs.js`

**内容：** 直接使用讲义 Day09 提供的完整代码（第 2549-2728 行），包含 `CameraWs` 类和 `createCameraWs()` 工厂函数。

**修改文件：** `frontend/src/views/CameraDetectionPage.vue`

**修改内容：** 删除内嵌的 WebSocket 逻辑（`connectWebSocket`、`sendSingleFrame`、`renderAnnotatedFrame` 等内嵌函数），改为导入 `createCameraWs` 并调用其 API。具体实现可参考讲义第 2947-3117 行的 `DetectionPage.vue` 完整代码。

## 六、测试总结

| 模块 | 状态 | 问题等级 | 说明 |
|------|------|----------|------|
| 智能对话 | 正常 | — | 功能正常 |
| 检测工作台 | 基本正常 | — | 功能正常 |
| 摄像头检测 | 异常 | P0 | 3 个代码问题导致完全无法使用 |

**核心结论：** 摄像头无法连接的根本原因是实际代码在多处偏离了讲义标准：

1. Store 的 token key 从讲义的 `rsod_token` 被改为 `token`，但摄像头页面仍按讲义读取 `rsod_token`（取到 null）
2. 后端 WebSocket 比讲义多了 JWT 认证，但前端未适配（URL 未传 token）
3. 后端的配置确认消息类型从讲义的 `config_ok` 被改为 `config_ack`，前端仍按讲义监听 `config_ok`

按照讲义代码修复上述 3 个问题后，摄像头检测功能应可正常工作。同时建议按讲义创建 `cameraWs.js` 工具文件，使代码结构符合讲义的模块化设计。
