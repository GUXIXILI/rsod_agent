### BUG-001：Agent对话失败（LLM 502 Bad Gateway）

| 项目 | 内容 |
|------|------|
| **优先级** | P0 |
| **严重程度** | 严重 |
| **状态** | 未修复 |
| **模块** | 智能对话 / Agent |
| **发现轮次** | 第一轮 |

**现象描述**:
发送消息后AI无回复，智能对话功能完全不可用。

**复现步骤**:
1. 登录系统，进入智能对话页面
2. 发送任意消息（如"你好"）
3. 等待AI回复

**预期结果**: AI正常回复用户消息

**实际结果**: AI无回复，SSE流中断

**错误日志**:
```
2026-07-18 11:53:49 | ERROR | app.agent.detection_agent:chat_stream:213 |
Agent 流式执行异常: <html>
<head><title>502 Bad Gateway</title></head>
<body>
<center><h1>502 Bad Gateway</h1></center>
<hr><center>nginx</center>
</body>
</html>
```

**推断原因**:
外部LLM API服务返回502 Bad Gateway，属于第三方服务问题。可能是：
1. LLM服务暂时不可用
2. API密钥/额度问题
3. 网络连接问题

**影响范围**:
- 智能对话功能
- Agent工具调用
- 记忆增强功能
- RAG问答功能

### BUG-006：工具返回格式非JSON

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **严重程度** | 一般 |
| **状态** | 未修复 |
| **模块** | Agent / 工具调用 |
| **发现轮次** | 第一轮 |

**现象描述**:
Agent工具调用返回结果时，前端解析失败，控制台报错。

**错误信息**:
```
[tool_end] 解析失败: SyntaxError: Unexpected token '【'
[tool_result] 解析失败
[warn] [tool_end] JSON解析失败，以纯文本展示
```

**复现步骤**:
1. 进入智能对话页面
2. 发送触发工具调用的消息（如"系统有多少用户"）
3. 查看浏览器控制台

**预期结果**: 工具返回结果为标准JSON格式，前端正常解析展示

**实际结果**: 工具返回结果使用中文括号【】格式，JSON解析失败

**推断原因**:
Agent工具的返回格式未标准化，后端或工具实现返回了非JSON格式的字符串。

**影响范围**:
- 工具调用结果展示不规范
- 前端有降级处理（纯文本展示），不影响基本功能使用

### BUG-007：会话列表401错误

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **严重程度** | 轻微 |
| **状态** | 待确认 |
| **模块** | 智能对话 / 会话管理 |
| **发现轮次** | 第一轮 |

**现象描述**:
前端进入智能对话页面时，控制台报401错误，会话列表无法加载。

**错误信息**:
```
获取会话列表失败: AxiosError: Request failed with status code 401
```

**复现步骤**:
1. 登录系统
2. 进入智能对话页面
3. 查看浏览器控制台和左侧会话列表

**预期结果**: 会话列表正常显示历史会话

**实际结果**: 会话列表为空，控制台报401

**API验证**:
直接调用 `GET /api/chat/sessions` 接口返回正常（200），共49条会话记录。

**推断原因**:
前端请求会话列表时Token未正确携带，可能是：
1. 请求拦截器逻辑问题
2. Token过期但未刷新
3. 请求时机问题（页面加载时Token尚未设置）

### BUG-008：Element Plus Radio废弃警告

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **严重程度** | 轻微 |
| **状态** | 未修复 |
| **模块** | 前端组件 |
| **发现轮次** | 第一轮 |

**现象描述**:
使用el-radio组件的页面控制台出现废弃警告。

**警告信息**:
```
[warn] ElementPlusError: [el-radio] [API] label act as value is about to be
deprecated in version 3.0.0, please use value instead.
```

**复现步骤**:
1. 进入摄像头检测页面或其他使用Radio的页面
2. 查看浏览器控制台

**预期结果**: 无废弃警告

**实际结果**: 出现废弃警告

**推断原因**:
Element Plus Radio组件使用了label属性作为值，在3.0版本中将废弃，应改用value属性。

**影响范围**:
- 功能正常，仅控制台警告
- 未来升级Element Plus版本时可能出现兼容问题

### BUG-009：视频检测失败 - MinIO 431错误

| 项目 | 内容 |
|------|------|
| **优先级** | P1 |
| **严重程度** | 严重 |
| **状态** | 新发现 |
| **模块** | 视频检测 |
| **发现轮次** | 第三轮 |

**现象描述**:
上传视频进行检测时，任务提交成功但处理失败，状态为failed。

**复现步骤**:
1. 进入视频检测页面
2. 上传MP4视频文件
3. 等待检测完成
4. 查看检测结果

**预期结果**: 视频检测完成，返回检测结果和标注视频

**实际结果**: 视频检测失败，进度卡在10%，状态显示unknown

**错误日志**:
```
2026-07-18 12:03:41 | ERROR |
app.services.video_task_service:_run_video_detection:156 |
视频异步任务失败: task_id=183, error=non-XML response from server;
Response code: 431, Content-Type: text/plain; charset=utf-8,
Body: 431 Request Header Fields Too Large

File "D:\clone\rsod_agent\backend\app\services\detection_service.py",
line 372, in detect_video
    video_url = minio_client.upload_bytes(video_bytes, object_name, f"video/{ext}")
```

**错误位置**:
- 文件：`app/services/detection_service.py`
- 行号：372行
- 函数：`detect_video` → `minio_client.upload_bytes()`

**推断原因**:
MinIO服务器返回431 Request Header Fields Too Large错误。可能原因：
1. MinIO配置的请求头大小限制过小
2. 上传大文件时请求头超过MinIO的限制
3. MinIO客户端配置问题

**影响范围**:
- 视频检测功能完全不可用
- 所有需要上传视频到MinIO的功能受影响

### BUG-010：个人设置缺少更新资料API

| 项目 | 内容 |
|------|------|
| **优先级** | P2 |
| **严重程度** | 一般 |
| **状态** | 新发现 |
| **模块** | 个人设置 |
| **发现轮次** | 第三轮 |

**现象描述**:
用户无法修改个人资料（昵称、邮箱、电话等）。

**复现步骤**:
1. 调用 PUT /api/user/profile 接口
2. 传入更新参数：{nickname, phone, email}

**预期结果**: 更新成功，返回更新后的用户信息

**实际结果**: 返回405 Method Not Allowed

**API现状**:
- ✅ GET /api/user/profile - 获取个人资料
- ❌ PUT /api/user/profile - 更新个人资料（不存在）
- ✅ PUT /api/user/password - 修改密码

**推断原因**:
后端只实现了获取个人资料和修改密码的接口，缺少更新个人基本信息的API。

**影响范围**:
- 用户无法修改昵称、邮箱、电话等个人信息
- 个人设置页面的编辑功能可能不可用
