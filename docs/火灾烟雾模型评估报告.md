# 火灾烟雾模型评估报告

## 1. 评估结论

当前推荐将 640 输入尺寸的 YOLO11n 模型作为服务器默认部署模型，将
960 输入尺寸模型保留为实验模型。

960 模型在验证集上的整体 mAP50 和 mAP50-95 仅比 640 模型高约
0.001 和 0.002，火焰召回率提高约 0.005，但输入像素数量是 640 模型的
2.25 倍。对于 CPU 资源有限的服务器，这部分精度增益不足以抵消额外的
推理开销。

## 3. 验证集结果

验证集包含 3,099 张图片和 3,931 个目标实例。

| 模型 | Precision | Recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: |
| YOLO11n 640 | 0.765 | 0.683 | 0.760 | 0.444 |
| YOLO11n 960 | 0.763 | 0.679 | 0.761 | 0.446 |

960 模型训练结束时的分类别结果：

| 类别 | Precision | Recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: |
| fire | 0.723 | 0.620 | 0.708 | 0.389 |
| smoke | 0.792 | 0.742 | 0.814 | 0.503 |

## 4. 独立评估验证

960 模型已通过独立评估 CLI 重新评估，评估过程成功生成：

- `evaluation.json`
- `confusion_matrix.png`
- `confusion_matrix_normalized.png`
- `P_curve.png`
- `R_curve.png`
- `F1_curve.png`
- `PR_curve.png`
- 验证集标签与预测样例图

执行命令：

```bash
python tools/evaluate_fire_smoke_model.py \
  --model path/to/best.pt \
  --data path/to/data.yaml \
  --output path/to/evaluation_output \
  --split val \
  --imgsz 960 \
  --batch 8 \
  --device 0
```

独立评估输出的模型 SHA256 与本报告记录一致，说明评估结果与指定权重文件
能够一一对应。

## 5. ONNX 导出验证

640 模型已通过项目模型导出服务真实导出为 ONNX：

- ONNX 版本：1.17.0
- ONNXScript 版本：0.6.0
- opset：18
- 导出文件大小：10,940,704 bytes
- `onnx.checker.check_model`：通过

PyTorch 2.12 的 ONNX 导出器需要 ONNXScript。项目依赖已显式锁定兼容
版本，并将导出 opset 固定为 18，避免低 opset 自动转换失败。

## 6. 部署建议

1. 默认上传和注册 640 模型的 `best.pt`。
2. CPU 推理服务默认使用 `imgsz=640`。
3. 需要跨框架部署时使用已验证的 `best.onnx`。
4. 960 模型不要覆盖 640 模型，作为独立版本保留，供后续 GPU 部署或火焰
   小目标专项测试使用。
5. 正式切换模型前，在服务器真实视频流上复测延迟、CPU 占用、误报率和
   漏报率。
