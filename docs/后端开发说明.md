# 火灾烟雾智能检测平台 — 后端开发说明

## 1. 项目概述

- **项目名称**：火灾烟雾智能检测平台（Fire & Smoke Detection Platform）
- **后端技术栈**：FastAPI + SQLAlchemy + Alembic + PostgreSQL + Redis + MinIO + YOLOv11 + DashScope LLM（通义千问）
- **后端完成度**：约 98%
- **分层架构**：

```
API 层（app/api/）
   ↓ 调用
Service 层（app/services/）
   ↓ 操作
Entity 层（app/entity/）— ORM 模型 + Pydantic Schema
   ↓ 持久化
Database 层（app/database/）— SQLAlchemy Session
```

## 6. LLM 对话功能

### 集成方式

通过 `langchain-openai` 的 `ChatOpenAI` 对接阿里云 DashScope（通义千问），兼容 OpenAI API 协议。

### 端点

- **非流式**：`POST /api/chat/messages` — 发送消息，返回完整 AI 回复
- **流式 SSE**：`POST /api/chat/messages/stream` — 返回 `text/event-stream`，前端逐 token 渲染

### 配置

在 `.env` 中修改：

```
QWEN_API_KEY=sk-xxxxx
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen-plus
```

### 上下文加载

对话服务自动加载最近 10 条历史消息作为上下文传入 LLM，保证对话连贯性。逻辑在 `app/services/chat_service.py` 中实现。

## 8. 开发注意事项

1. **`.env` 不入库** — 新成员必须从 `.env.example` 复制，不要将 `.env` 提交到 Git。
2. **MINIO_BUCKET 必须为 `fire-detection-images`** — 代码中硬编码了此桶名，修改会导致文件上传/下载失败。
3. **APP_NAME 必须为 `Fire & Smoke Detection Platform`** — 与前端约定一致，勿随意修改。
4. **JWT_SECRET_KEY** — 非 DEBUG 模式下长度必须 ≥ 32 字符，否则服务拒绝启动。
5. **审计日志中间件已注册** — 所有 POST/PUT/DELETE 请求自动记录到 `operation_logs` 表，无需手动调用。
6. **定时任务** — 每天 03:00 自动清理过期 failed 状态的检测任务，需在 `.env` 中设置 `SCHEDULER_ENABLED=true` 启用。
7. **登录速率限制** — 连续登录失败会触发临时锁定（默认锁定时间由 `LOGIN_LOCKOUT_MINUTES` 配置）。
8. **CORS** — `ALLOWED_ORIGINS` 中配置前端地址，多个地址用逗号分隔。
