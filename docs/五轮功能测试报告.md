## 九、重点BUG详细说明

### BUG-001：LLM 400错误（function.arguments为None）【P0 - 未修复】

**错误现象**：
- 发送消息触发工具调用后，tool_start和tool_end事件正常
- 但随后触发error事件，错误信息为：`Agent 执行出错：Error code: 400 - function.arguments为None`
- 对话无法正常完成，无最终AI回复

**推断原因**：
- LangGraph Agent在处理工具调用结果时，向LLM发送的function call消息中arguments字段为None
- LLM API（兼容OpenAI格式）要求function.arguments必须是字符串，不能为null/None
- 可能是工具返回结果解析时出现问题，或Agent状态管理中arguments字段丢失

**影响范围**：
- 所有需要LLM生成最终回复的对话场景
- RAG问答功能
- 多轮对话记忆功能
- 所有Agent工具调用后的总结回复

---

### BUG-015：nickname更新不持久化【P1 - 新发现】

**错误现象**：
- PUT /api/user/profile 更新nickname为"test_r5"
- PUT响应中data.nickname = "test_r5"（返回成功）
- 立即GET /api/user/profile，返回nickname = None
- email和phone字段更新正常

**推断原因**：
- 后端更新逻辑中可能遗漏了nickname字段
- 或数据库users表中nickname字段与ORM模型字段名不匹配
- 或PUT响应是构造的（回显请求值），未真正写入数据库
- 或存在缓存层，GET从缓存读取旧值

---

### BUG-016：非图像文件不拒绝【P1 - 新发现】

**错误现象**：
- 上传txt文件到POST /api/detection/single
- 返回200 OK，message="检测完成"
- fire_level=None，detection_count=0

**推断原因**：
- 检测接口缺少文件类型校验
- 未验证文件扩展名和MIME类型是否为支持的图像格式
- YOLO模型可能对非图像文件返回空结果，但接口层未做前置校验

**风险**：
- 用户可能上传恶意文件
- 浪费YOLO推理资源
- 返回结果可能误导用户

---

### BUG-017：不存在的scene_id返回500【P2 - 新发现】

**错误现象**：
- 传入scene_id=2（数据库中只有scene_id=1）
- 返回500 Internal Server Error
- 错误信息：`ForeignKeyViolation: insert or update on table "detection_tasks" violates foreign key constraint`

**推断原因**：
- 业务层缺少scene_id存在性校验
- 直接将scene_id写入数据库，由数据库外键约束拦截
- 应在业务层先查询scene是否存在，不存在返回400或404
