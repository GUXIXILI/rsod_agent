## P0 致命缺陷

### BUG-001：Agent 对话执行失败 - LLM API 返回 400 错误

**所属模块**：Day08 / Day11 - 智能对话 / Agent 核心功能
**严重等级**：P0 致命
**影响范围**：所有基于 Agent 对话的功能

#### 缺陷现象
在对话页面发送任何消息（包括纯文本问答），AI 回复失败，显示错误信息：
```
Agent 执行出错：Error code: 400 - {'error': {'message': '3 validation errors for ValidatorIterator
0.ChatCompletionMessageFunctionToolCallParam.function.arguments
 Input should be a valid string
```

#### 复现步骤
1. 登录系统，进入对话页面
2. 在输入框输入任意消息（如"你好"）
3. 点击发送按钮
4. 等待 AI 回复

#### 预期结果
- AI 正常回复，流式显示文本
- 若需要调用工具，正确显示工具调用状态和结果

#### 实际结果
- 对话中断，显示红色错误提示
- SSE 连接异常终止（net::ERR_ABORTED）
- 前端控制台显示 400 错误

#### 错误原因推断
**推断原因**：LangChain Agent 在构造 tool call 消息时，`function.arguments` 字段为 `None` 而非 JSON 字符串，导致 LLM API 校验失败。

可能的具体原因：
1. Agent 工具定义有问题，返回的 tool_call 参数格式不正确
2. LangChain 版本与 LLM API 不兼容，导致消息格式转换错误
3. 某个工具的返回值为 None，在序列化时没有正确处理
4. `query_user_list` 等工具被错误调用，且返回格式不符合 OpenAI tool call 规范

#### 相关证据
- 前端错误：`Agent 执行出错：Error code: 400`
- 控制台错误：`net::ERR_ABORTED http://localhost:5173/api/chat/messages/stream`
- 错误详情：`function.arguments Input should be a valid string`

### BUG-002：数据看板页面完全空白

**所属模块**：Day10 - 数据看板
**严重等级**：P0 致命
**影响范围**：数据统计展示

#### 缺陷现象
进入数据看板页面，仅显示页面标题"数据看板"，下方内容区域完全空白，没有统计卡片、图表等任何内容。

#### 复现步骤
1. 登录系统
2. 点击侧边栏"数据看板"菜单
3. 观察页面内容

#### 预期结果
- 显示 4 个统计卡片（总检测次数、检出火焰次数、检出烟雾次数、触发预警次数）
- 显示火情等级分布图（饼图/环形图）
- 显示检测趋势折线图
- 显示类别分布、场景分布、任务类型分布图表

#### 实际结果
- 页面只有标题"数据看板"
- 没有任何统计卡片或图表
- 页面无 JS 报错（控制台无新错误）

#### 错误原因推断
**推断原因**：前端 DashboardPage.vue 组件渲染逻辑有问题。

可能的具体原因：
1. 组件模板中条件渲染错误（如 v-if 条件不满足）
2. API 调用路径错误，数据未正确获取
3. ECharts 图表组件未正确导入或初始化
4. 响应式数据定义错误，导致页面不渲染
5. 后端 API 返回数据格式与前端期望不匹配（但 API 本身 200 正常返回）

#### 相关证据
- 页面快照：仅包含标题，无内容区元素
- API 验证：`/api/stats/overview` 等 6 个统计接口均正常返回数据
- 控制台：无新增 JS 错误

## P1 严重缺陷

### BUG-003：检测结果 annotated_url 为 null

**所属模块**：Day08 - 单图/批量/ZIP 检测
**严重等级**：P1 严重
**影响范围**：检测结果展示

#### 缺陷现象
单图检测完成后，返回的检测任务中 `annotated_url` 字段为 `null`，导致前端无法展示标注后的图片。

#### 复现步骤
1. 调用单图检测 API：`POST /api/detection/single`
2. 传入图片文件和场景参数
3. 获取检测结果，查看 `annotated_url` 字段

#### 预期结果
- `annotated_url` 返回标注图的 MinIO 访问 URL
- 前端可展示标注了边界框的图片

#### 实际结果
- `annotated_url` 为 `null`
- `original_url` 也为 `null`
- 检测结果数据（目标、置信度）正常返回

#### 错误原因推断
**推断原因**：标注图生成后未正确上传到 MinIO，或数据库字段未更新。

可能的具体原因：
1. 标注图生成代码执行失败（如 OpenCV 绘制边界框出错）
2. MinIO 上传失败（Bucket 不存在、凭证错误、网络问题）
3. 数据库更新时遗漏了 annotated_url 字段
4. 标注图生成了但保存路径错误，导致上传失败
5. 检测服务与 MinIO 的连接配置有问题

#### 相关证据
- API 响应：`"annotated_url": null`
- 任务详情 API：`"annotated_url": null, "original_url": null`
- MinIO 服务状态：正常运行（healthy）

**影响范围**：用户信息展示与修改

#### 缺陷现象
进入个人设置页面，显示"暂无个人信息"，但实际上用户信息存在且 API 能正常返回。

#### 复现步骤

3. 查看个人信息区域

### BUG-005：RAG 知识库搜索无结果

**所属模块**：Day11 - RAG 知识库
**严重等级**：P1 严重
**影响范围**：知识问答功能

#### 缺陷现象
知识库统计显示有 15769 条记录，但搜索常见问题（如"什么是IoU"）返回 0 条结果。

#### 复现步骤
1. 调用知识库搜索 API：`GET /api/knowledge/search?q=什么是IoU`
2. 查看返回的 results 数组

#### 预期结果
- 返回与问题相关的知识库片段
- results 数组包含匹配的文档内容
- count > 0

#### 实际结果
- `results` 为空数组
- `count` 为 0
- API 状态码 200，无报错

#### 错误原因推断
**推断原因**：向量搜索匹配度低或索引构建有问题。

可能的具体原因：
1. Embedding 模型问题：中文语义理解能力差，导致 query 向量与文档向量不匹配
2. 向量索引未正确构建：知识库文档的向量未写入 pgvector
3. 搜索阈值设置过高：相似度阈值太高，过滤了所有结果
4. 中文分词问题：query 没有正确分词，影响向量质量
5. 知识库文档内容与搜索词不相关（但"IoU"是目标检测基础概念，应该有相关内容）
6. 知识库统计数据不准确，实际向量记录数少于显示值

#### 相关证据
- 知识库统计：`total_records: 15769`
- 搜索结果：`results: [], count: 0`
- 搜索词："什么是IoU"（目标检测领域基础概念）

### BUG-006：工具返回格式错误 - 前端 JSON 解析失败

**所属模块**：Day11 - Agent 优化 / SSE 流式增强
**严重等级**：P1 严重
**影响范围**：流式对话体验、工具调用展示

#### 缺陷现象
前端控制台显示 tool_end / tool_result 解析失败错误，错误信息为：
```
[tool_end] 解析失败: SyntaxError: Unexpected token '【', "【用户列表】（共 1"... is not valid JSON
```

#### 复现步骤
1. 在对话页面发送消息
2. Agent 调用用户列表查询工具
3. 查看浏览器控制台

#### 预期结果
- 工具返回 JSON 格式数据
- 前端正确解析并展示工具调用结果
- tool_end 事件包含结构化的工具返回值

#### 实际结果
- 工具返回中文文本格式（如 `【用户列表】（共 106 人）1. testuser...`）
- 前端尝试 JSON.parse 失败
- 控制台报错，工具结果无法正常展示

#### 错误原因推断
**推断原因**：Agent 工具的返回格式与前端解析逻辑不匹配。

可能的具体原因：
1. 工具函数返回的是格式化的中文文本字符串，而非 JSON 对象
2. 前端 stream.js 中 tool_end 事件处理逻辑期望 JSON 格式
3. LangChain 工具定义中没有正确设置返回值 schema
4. Agent 在调用工具后对返回结果做了额外的文本格式化处理
5. 前后端约定的 SSE 事件数据格式不一致

#### 相关证据
- 控制台错误：`[tool_end] 解析失败: SyntaxError: Unexpected token '【'`
- 错误内容：工具返回了 `【用户列表】（共 106 人）...` 格式的文本
- 涉及文件：`src/utils/stream.js`、`src/views/ChatPage.vue`

## P2 一般缺陷

### BUG-007：会话列表获取报 401 错误

**所属模块**：Day08 - 会话管理
**严重等级**：P2 一般
**影响范围**：对话页面加载体验

#### 缺陷现象
页面加载时，控制台显示"获取会话列表失败: AxiosError: Request failed with status code 401"错误。

#### 复现步骤
1. 登录后进入对话页面
2. 打开浏览器控制台
3. 刷新页面

#### 预期结果
- 会话列表正常加载
- 无 401 错误

#### 实际结果
- 控制台显示 401 错误
- 但会话列表最终还是能显示（可能重试成功或有其他获取方式）

#### 错误原因推断
**推断原因**：页面初始化时获取会话列表的时机过早，Token 尚未注入请求拦截器。

可能的具体原因：
1. 组件 mounted 时立即请求会话列表，但 Pinia 中的 token 还没就绪
2. 请求拦截器的 token 注入逻辑有竞态条件
3. 页面刷新后，从 localStorage 恢复 token 的时机晚于 API 调用
4. 有重复的请求，第一次失败后自动重试成功

#### 相关证据
- 控制台错误：`获取会话列表失败: AxiosError: Request failed with status code 401`
- 来源文件：`src/stores/agent.js:36`
- 最终会话列表可正常显示

### BUG-008：Element Plus Radio 组件废弃警告

**所属模块**：前端组件库
**严重等级**：P2 一般
**影响范围**：控制台警告（不影响功能）

#### 缺陷现象
控制台显示 Element Plus 废弃警告：
```
ElementPlusError: [el-radio] [API] label act as value is about to be deprecated in version 3.0.0, please use value instead.
```

#### 复现步骤
1. 进入使用了 el-radio 组件的页面（如设置页面）
2. 查看控制台警告

#### 预期结果
- 无废弃 API 警告
- 使用最新推荐的 API 写法

#### 实际结果
- 控制台显示废弃警告
- 功能正常但有潜在升级风险

#### 错误原因推断
**推断原因**：项目中使用的 el-radio 组件仍在使用 label 属性作为值，而新版本 Element Plus 推荐使用 value 属性。

#### 相关证据
- 控制台警告：`[el-radio] [API] label act as value is about to be deprecated`
- 涉及组件：el-radio / el-radio-group
