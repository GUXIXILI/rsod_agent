## 一、未修复缺陷（6个）

### BUG-001：LLM返回400错误 - function.arguments为None
- **优先级**：P0
- **模块**：智能对话 / Agent
- **错误现象**：
  - 触发工具调用后，tool_start和tool_end事件正常
  - 随后触发error事件，内容为：`Agent 执行出错：Error code: 400 - function.arguments为None`
  - 对话无法正常完成，无最终AI回复
  - 简单问候（不触发工具）也可能因SSE流异常导致超时
- **推断原因**：
  - LangGraph Agent在处理工具调用结果时，向LLM发送的function call消息中arguments字段为None
  - LLM API要求function.arguments必须为字符串类型，不能为null/None
  - 可能是Agent状态管理中工具调用参数丢失，或工具结果格式转换有误
- **影响范围**：所有需要LLM生成最终回复的对话场景（普通对话、RAG问答、多轮对话、工具调用总结）
- **验证方法**：POST /api/chat/messages/stream 发送"系统有多少个用户"，监听SSE事件，观察是否有error事件
- **发现轮次**：第一轮
- **最近验证**：第五轮（2026-07-18）- 仍存在

### BUG-012：用户列表page_size参数未生效
- **优先级**：P2
- **模块**：用户管理
- **错误现象**：
  - 请求 `GET /api/user/list?page=1&page_size=5`
  - 实际返回20条数据，而非预期的5条
- **推断原因**：
  - 后端分页参数解析有误，可能使用了默认值覆盖了请求参数
  - 或参数名不一致（如应为pageSize而非page_size）
- **影响范围**：用户列表分页功能异常，前端分页控件显示不正确
- **验证方法**：对比不同page_size值的返回结果数量
- **发现轮次**：第四轮
- **最近验证**：第五轮（2026-07-18）- 仍存在

### BUG-015：PUT更新nickname后GET返回None【新发现】
- **优先级**：P1
- **模块**：用户管理 / 个人设置
- **错误现象**：
  1. PUT /api/user/profile 请求body: `{nickname: "test_r5", email: "test@test.com", phone: "13800000000"}`
  2. PUT响应data中nickname = "test_r5"（看起来成功了）
  3. 立即调用 GET /api/user/profile，返回nickname = None
  4. 但email和phone字段更新正常（GET能读到新值）
- **推断原因**：
  - 后端更新逻辑中遗漏了nickname字段的SQL更新
  - 或数据库表中nickname字段名与ORM模型不匹配
  - 或PUT响应是直接回显请求参数，未真正写入数据库
  - 或存在缓存层，GET从缓存读取旧值（但email/phone正常，此可能性低）
- **影响范围**：用户昵称无法正常修改，个人设置页面昵称修改功能无效
- **验证方法**：
  1. GET /api/user/profile 记录原nickname
  2. PUT /api/user/profile 更新nickname
  3. 再次GET /api/user/profile 验证nickname是否更新
- **发现轮次**：第五轮
- **最近验证**：第五轮（2026-07-18）- 新发现

### BUG-016：非图像文件上传检测不拒绝【新发现】
- **优先级**：P1
- **模块**：图像检测
- **错误现象**：
  - 上传txt文本文件到 POST /api/detection/single
  - 接口返回200 OK，message="检测完成"
  - fire_level=None，fire_object_count=0，smoke_object_count=0
- **推断原因**：
  - 检测接口缺少文件类型前置校验
  - 未验证文件扩展名和MIME类型是否为支持的图像格式（jpg/jpeg/png等）
  - YOLO模型对非图像文件可能返回空结果，但接口层未做拦截
- **影响范围**：
  - 用户可能上传非图像文件得到误导性结果
  - 浪费YOLO推理资源
  - 存在安全风险（恶意文件上传）
- **验证方法**：上传txt/pdf等非图像文件，观察是否返回400错误
- **发现轮次**：第五轮
- **最近验证**：第五轮（2026-07-18）- 新发现

### BUG-017：不存在的scene_id返回500错误【新发现】
- **优先级**：P2
- **模块**：图像检测 / 场景管理
- **错误现象**：
  - 传入scene_id=2（数据库中只有scene_id=1, name=rsod）
  - 接口返回500 Internal Server Error
  - 错误信息：`ForeignKeyViolation: insert or update on table "detection_tasks" violates foreign key constraint "detection_tasks_scene_id_fkey"`
- **推断原因**：
  - 业务层缺少scene_id存在性校验
  - 直接将scene_id写入detection_tasks表，由数据库外键约束拦截
  - 应在业务逻辑层先验证scene是否存在
- **影响范围**：
  - 用户选择不存在的场景时得到不友好的500错误
  - 前端无法正确处理该错误场景
  - 暴露数据库错误信息，存在信息泄露风险
- **验证方法**：
  1. GET /api/scenes 获取可用场景列表
  2. POST /api/detection/single 传入不存在的scene_id
  3. 观察是否返回400/404而非500
- **发现轮次**：第五轮
- **最近验证**：第五轮（2026-07-18）- 新发现
