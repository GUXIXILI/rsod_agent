## 五、未来要实现的功能

　　当前主干虽然已经能跑起来，而且 11170eb 吸收提交后功能进一步完善，但离最终目标还有一段距离，下面列出未来要实现的功能方向，供组员认领。

　　第一是多 agent 工作室模式。这是前端的重头戏，目标是做一个类似豆包的统一入口页面，一个输入框，用户输入任何内容，由管理 agent 来判断内容类型，再由管理 agent 自行调用其余 agent 完成工作，给用户一条龙服务的感觉。目前后端的对话模块已经能跑通流式对话，但多 agent 调度逻辑还没开始，前端统一入口页面也还没做。

　　第二是摄像头实时检测。11170eb 吸收提交后前端已有 CameraDetectionPage.vue 摄像头检测页面（路由 /camera-detection），与 DetectionPage.vue 双版本共存，但后端摄像头接入层尚未实现。最终目标就是接入摄像头实时监控，后端需要新建 app/camera 模块，处理 RTSP/RTMP 流接入、逐帧推理、跳帧优化、结果回传、WebSocket 推送、断线重连等环节。

　　第三是检索增强知识库。也就是 RAG 消防知识问答，让 agent 能基于消防知识库回答用户问题。PostgreSQL 已经带了 pgvector 扩展，向量检索的基础设施是现成的，剩下的是知识库构建、向量化、检索召回这套流程。

　　第四是容器化部署完善。当前 docker-compose 已经能起全栈，但还有一些细节要打磨，比如端口 8000 可能被本机其它服务占用需要临时换 8001、GPU 加速配置还是注释状态、Ollama 本地大模型服务还是可选未启用等。这些都需要在实际部署中逐个验证。

　　第五是前端界面美化。当前前端页面功能基本齐全，但视觉风格还比较朴素，距离豆包那种顺滑的交互体验还有差距，需要前端人员持续打磨。

## 八、最终功能目标

　　目前这个项目的最终功能目标是：用户给一个摄像头，这个东西接入摄像头，然后这个项目实时监控，查看是否有火灾隐患。换句话说，最终要交付的不是只能在网页上传图片做单图检测的演示demo，而是能接到真实摄像头视频流上、7 乘 24 小时不间断监测画面中火焰和烟雾、一旦发现异常就自动预警的监控系统。这个目标决定了后续所有开发工作的优先级，摄像头实时检测是必须啃下来的硬骨头，预警分级和推送也要跟上。

　　关于这个最终目标的详细背景，包括项目定位、目标用户、使用场景、数据集可行性、技术可行性、商业可行性、15 天实施计划、风险评估等完整内容，可以参考 d:\clone\技术路线\火灾烟雾智能检测系统-需求与可行性分析.md 这个文件。文件里写清楚了短期目标、中期目标、长期目标三档，短期是可演示原型，中期加实时摄像头检测和智能体对话，长期做产品化。组员做开发决策的时候建议先把这个文件读一遍，对齐目标再动手。

## 九、实现方式简略说明

　　关于如何实现最终功能，这里做一些简略说明，但总体让大家自由实现，不强制具体方式。实现思路上，检测这一层基于 YOLOv11 检测模型，主干里已经有 fire_smoke_yolo11n_v1 这个训练好的权重，后端 fire_smoke_detection_service 负责加载模型做推理，这一层是现成的，摄像头实时检测可以在这套推理能力上扩展，重点解决视频流解码和逐帧推理的性能问题，比如跳帧检测、跟踪算法辅助、必要时上 GPU 加速。11170eb 吸收提交后视频检测 API（detectVideo、getVideoStatus）已经可用，可以作为摄像头实时检测的参考实现。

　　调度这一层走多 agent 架构，管理 agent 负责理解用户意图、分发任务给检测 agent、问答 agent、分析 agent 等子 agent，子 agent 各司其职把结果回传给管理 agent 汇总。后端对话模块已经把流式通信通道打通，接下来是把 agent 调度逻辑填进去。实时流处理这一层，摄像头视频流进来后要边解码边推理边回传，不能等整段视频读完再处理，这就涉及流式管道的设计。

　　模型评估与导出这一层，11170eb 吸收提交后已经基本完成，包括 model_evaluation_service、model_artifact_service、evaluate_fire_smoke_model 评估工具、3 个新路由（评估、导出、下载）、5 个 Pydantic schema 类、onnx 与 onnxscript 依赖、6 个测试文件。后续只需在前端补对应的评估展示页面和导出下载入口，并完善 ONNX 导出能力的实际验证。

　　以上只是思路层面的简略说明，具体用什么模型、怎么调度、流处理怎么实现，由各成员自由发挥，只要最终能跑通摄像头实时检测加预警这条主线就行。鼓励大家在思路上多交流，但实现细节不强求统一。
