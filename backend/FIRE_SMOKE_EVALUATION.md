## 4. 推荐阈值

### 平衡模式

用于单图检测或尚未实现视频时序确认的接口：

```text
fire:  0.25
smoke: 0.20
```

- Fire Precision 0.654、Recall 0.668、负样本报警率 0.91%。
- Smoke Precision 0.701、Recall 0.809、负样本报警率 4.54%。

### 高召回告警模式

只有在后端实现连续多帧确认、告警冷却和重复框合并后使用：

```text
fire:  0.20
smoke: 0.20
```

Fire Recall 可提高到 0.706，但 Precision 降为 0.593，不能直接用单帧结果触发正式告警。

## 5. 错误样本观察

1. Fire 的主要问题是漏检小火点、远距离火焰、夜间火场和被烟雾遮挡的火焰。
2. 数据集常把连续火场标注为多个小框，模型则预测一个较大的合并框。IoU 匹配会同时记录 FP 和 FN，因此部分“误报”是标注粒度与模型框粒度不一致，并非模型在无火区域识别出火焰。
3. 纯负样本上的 fire 报警率明显低于全量 FP 比例，说明 fire 的主要矛盾仍是定位和漏检，而不是大量无火误报。
4. Smoke 指标已明显优于 fire，但低阈值下负样本报警率较高，需要视频时序过滤。

## 6. 验收结论

| 门槛 | 结果 |
| --- | --- |
| 算法基线 | 通过 |
| Smoke 集成候选 | 通过 |
| Fire 集成候选 | 未通过，Recall 仍不足 |
| 生产消防告警 | 不具备直接上线条件 |

下一阶段应先在推理服务中支持 fire/smoke 独立阈值和视频连续帧确认，再使用仓库、室内、夜间监控及强光干扰数据做专项评估。若专项数据仍显示 fire Recall 不足，再考虑补充数据、提高输入分辨率或训练 YOLO11s。
