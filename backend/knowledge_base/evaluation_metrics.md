# 模型评估指标详解

## 一、基础概念

### 1.1 混淆矩阵
混淆矩阵是评估分类模型性能的基础工具：

|           | 预测为正 | 预测为负 |
|-----------|---------|---------|
| 实际为正  | TP（真正例） | FN（假负例） |
| 实际为负  | FP（假正例） | TN（真负例） |

- TP（True Positive）：正确检测到的火灾/烟雾目标
- FP（False Positive）：误报，将非火灾/烟雾目标错误检测为火灾/烟雾
- FN（False Negative）：漏报，未能检测到实际存在的火灾/烟雾目标
- TN（True Negative）：正确识别为背景/非目标

### 1.2 精确率（Precision）
精确率衡量预测为正例的样本中真正为正例的比例：
- Precision = TP / (TP + FP)
- 高精确率意味着误报少
- 在火灾检测中，精确率低意味着频繁误报，影响用户体验

### 1.3 召回率（Recall）
召回率衡量实际为正例的样本中被正确预测的比例：
- Recall = TP / (TP + FN)
- 高召回率意味着漏报少
- 在火灾检测中，召回率低意味着漏报火灾，可能导致严重后果

### 1.4 F1-Score
F1-Score 是精确率和召回率的调和平均：
- F1 = 2 × (Precision × Recall) / (Precision + Recall)
- 综合考虑精确率和召回率
- 适用于类别不平衡的情况

## 二、目标检测特有指标

### 2.1 mAP（平均精度均值）
mAP 是目标检测领域最核心的评估指标：

**计算步骤：**
1. 对每个类别，根据置信度排序所有检测结果
2. 在不同置信度阈值下计算 Precision 和 Recall
3. 绘制 Precision-Recall 曲线
4. 计算曲线下面积（AUC）得到 AP
5. 对所有类别的 AP 取平均得到 mAP

**mAP 变体：**
- mAP@0.5：IoU 阈值为 0.5 时的 mAP，较宽松
- mAP@0.5:0.95（mAP@0.5:0.05:0.95）：在 0.5 到 0.95 之间以 0.05 为步长的多个 IoU 阈值下的平均 mAP，更严格
- mAP@0.75：IoU 阈值为 0.75 时的 mAP，更严格

### 2.2 IoU 阈值的影响
- 较低的 IoU 阈值（如 0.5）：允许较大位置偏差，mAP 通常较高
- 较高的 IoU 阈值（如 0.75）：要求更精确的定位，mAP 通常较低
- 火灾检测中，由于目标形状不规则，通常使用 mAP@0.5 作为主要指标

## 三、训练相关指标

### 3.1 损失函数
YOLO 模型的损失函数通常包含三个部分：

**边界框回归损失（Box Loss）：**
- 衡量预测框与真实框的位置差异
- 常用 CIoU（Complete IoU）损失，同时考虑重叠面积、中心点距离和宽高比

**分类损失（Class Loss）：**
- 衡量类别预测的准确性
- 通常使用二元交叉熵损失（BCE Loss）

**置信度损失（Objectness Loss）：**
- 衡量目标存在性预测的准确性
- 用于区分前景和背景

### 3.2 训练曲线分析
- 训练损失下降缓慢：可能需要调整学习率或增加数据量
- 验证损失上升：可能出现过拟合，需要正则化或早停
- 损失震荡：可能需要调整批量大小或学习率
- 损失不收敛：可能需要检查数据质量或调整模型结构

## 四、火灾检测特有指标

### 4.1 误报率（False Alarm Rate）
误报率是火灾检测系统的重要指标：
- 误报率 = FP / (FP + TN) = 误报次数 / 总非目标样本数
- 单位：误报次数/小时 或 误报次数/天
- 火灾检测系统要求极低的误报率，否则会影响用户信任

### 4.2 漏报率（Miss Rate）
漏报率衡量系统未能检测到真实火灾的概率：
- 漏报率 = FN / (TP + FN) = 1 - Recall
- 在火灾检测中，漏报可能造成严重后果，应尽可能降低

### 4.3 检测延迟（Detection Latency）
- 从火灾发生到系统检测到火灾的时间
- 包括图像采集、模型推理、后处理的总时间
- 早期检测至关重要，延迟越低越好

### 4.4 报警确认帧数
- 连续检测到火灾的帧数阈值
- 帧数太少：容易误报
- 帧数太多：延迟报警，可能错过最佳响应时间
- 通常设置为 3-5 帧作为平衡

### 4.5 综合评估策略
火灾检测模型的评估应综合考虑：
1. 检测精度：mAP、Precision、Recall
2. 检测速度：FPS（帧每秒）、推理延迟
3. 误报控制：误报率、报警确认机制
4. 鲁棒性：不同场景、光照、天气条件下的表现
5. 早期检测能力：小火苗、初期烟雾的检测能力
